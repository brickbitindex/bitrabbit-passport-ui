BitRabbit Passport UI

BitRabbit Passport规定了2种主题色：

  • 主题色1：#fc466b
  • 主题色2：#1a9be3

整个UI在设计之初考虑的就是使用2种主题色来进行互相的协调。在设计之初我们遵循3个原则：

  • 明确：页面元素的呈现、转化、反馈必须明确表达目的。
  • 深刻：页面元素的呈现、转化、反馈可以教育用户，使得用户能迅速适应页面。
  • 克制：页面元素在设计上必须减少表达，力求以最有效、简洁的方式进行呈现、转化、反馈。

对于接入方式的选择，我们做了很多设计，我们认为，好的情况应该如下：

  • 我们不愿意大量使用class，复杂定制会使得classList过长，难以辨认，所以对于组件，我们更倾向于使用class来定位元素，并使用属性来进行配置。
  • 我们觉得Bootstrap是个优秀的UI框架，但是复杂的组件会带有过多的DOM结构，增加了编码复杂性。所以我们设计时必须引入JS的帮助，使用明确的简单的模板，用JS来生成一些额外的DOM。
  • 在某些情况下，若是库本身没有加载成功，比如CSS和JS文件因为网络原因没有加载，那么我们必须保证页面的主逻辑是流畅的。
  • 我们希望，在库成功加载之后，用户的其他业务逻辑是不变的。例如对于一个id为select的select元素，我们进行了大量新的DOM生成，但是若是其他脚本中使用了$('#select').val()仍然能准确获得数据，即便页面上看起来已经没有了这个元素。

引入方式

在页面上引入JS和CSS模板：

<link rel="stylesheet" href="https://assets.zjzsxhy.com/g/bpc/0.4.10/bpc.min.css">
<script src="https://assets.zjzsxhy.com/g/bpc/0.4.10/bpc.min.js"></script>

若是通过了动态代码添加了新的模板，只需要在全局调用BPC.render重新渲染即可。
